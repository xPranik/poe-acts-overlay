# Трёхуровневая цветовая индикация таймера относительно PB

Дата: 2026-07-19

## Цель

Сейчас таймер красит `Δ` и общее время в 2 цвета: зелёный (не отстаём) /
красный (отстаём). Нужен третий, промежуточный уровень — оранжевый —
для случая «почти отстал», чтобы игрок видел приближение к потере рекорда
до того, как реально его потеряет.

## Логика

`delta = elapsed − pbCumulativeMs` (то же сравнение, что уже используется:
текущее накопленное время минус PB в той же точке маршрута — акте).

| Условие | Класс | Цвет |
|---|---|---|
| `delta ≤ 0` | `ahead` | зелёный `#6fce87` (уже есть) |
| `0 < delta ≤ 10_000` (10 сек) | `close` | оранжевый `#e0a955` (новый) |
| `delta > 10_000` | `behind` | красный `#e08585` (уже есть) |
| PB нет / delta `null` | без класса | нейтральный (как сейчас) |

Порог `CLOSE_THRESHOLD_MS = 10_000` — константа, не настройка пользователя.

## Где применяется

1. **`.timer-total`** (`Timer.tsx`) — общее время в развёрнутой панели
   таймера. Уже вызывает `deltaClass(rows[rows.length - 1]?.delta ?? null)`
   — заменить на трёхуровневую версию.
2. **`.split-delta`** (`Timer.tsx`, построчно в списке актов) — та же замена.
   `.split-delta.gold` (лучший сегмент) не меняется — это отдельная,
   ортогональная подсветка.
3. **`MiniTimer`** (`App.tsx`) — цифры в шапке при свёрнутой панели таймера.
   Сейчас компонент не знает о PB. Добавить вычисление `delta` из
   `timer.pb` (найти запись по `timer.currentAct`) и `liveElapsed(timer, now)`,
   применить тот же трёхуровневый класс. Если `pb` нет или для текущего
   акта нет записи — без класса (нейтральный цвет, как сейчас).
   В `paused`/`finished` цвет остаётся статичным (не пересчитывается по
   тикающему `now`, т.к. `liveElapsed` в этих статусах и так не зависит от
   `now`).

## Реализация

- Одна общая функция и константа в `Timer.tsx`, экспортируются рядом с
  `fmt`/`liveElapsed`:
  ```ts
  export const CLOSE_THRESHOLD_MS = 10_000

  export function deltaColorClass(delta: number | null): string {
    if (delta == null) return ''
    if (delta <= 0) return 'ahead'
    if (delta <= CLOSE_THRESHOLD_MS) return 'close'
    return 'behind'
  }
  ```
  Внутри `Timer.tsx` локальная `deltaClass` заменяется на вызов
  `deltaColorClass` (сигнатура использования не меняется — как и раньше,
  вызывается с `delta: number | null`).
- `App.tsx` импортирует `deltaColorClass` из `./Timer` и использует его в
  `MiniTimer` для вычисления класса.
- CSS (`styles.css`): добавить `.timer-total.close` и `.split-delta.close`
  рядом с существующими `.ahead`/`.behind` правилами, и `.mini-timer.close`
  / `.mini-timer.ahead` / `.mini-timer.behind` (у `MiniTimer` пока нет
  цветовых классов вообще, кроме `mini-timer-paused` для приглушения при
  паузе — это разные, независимые классы: `mini-timer-paused` управляет
  яркостью/тусклостью, а новый класс — оттенком по PB; они комбинируются).

## Крайние случаи

- Нет PB (`timer.pb === null`) — все три компонента остаются без цвета,
  как сейчас.
- Для текущего акта в PB нет записи (более длинный забег, где раньше PB
  не доходил) — `pbCum` не найден → `delta = null` → без класса.
- `idle` — `MiniTimer` не рендерится вовсе (уже так), цвет неприменим.
