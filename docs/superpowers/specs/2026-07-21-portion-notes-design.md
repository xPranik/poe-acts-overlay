# Заметки в блоке порции пресета (design)

## Проблема

Редактор пресета камней (`SettingsApp.tsx`, вкладка Presets → секция "Gem portions
(by quest)") позволяет для каждой прогрессивной порции (блок квеста вроде "Enemy at
the Gate") задать только `TAKE` (какой камень взять наградой) и `BUY` (что докупить
у торговца). Нет способа прикрепить произвольный свободный текст к порции — например
заметку про порядок действий, альтернативный камень под другую сборку и т.п.

## Решение

Добавить третье поле **NOTES** в блок порции — одно свободное текстовое поле,
хранящееся вместе с `take`/`buy` в том же `[[portion]]`. В оверлее заметка
показывается как обычный текст под заголовком порции, без чекбокса и без записи
в progress (аналогично `zone.notes` у обычной зоны).

Zone-gems список (записи зоны с выбором типа "Награда"/"Покупка" в выпадающем
списке, `EntryRow`) не затрагивается — это отдельный, не связанный с порциями UI.

## Модель данных

`src/shared/types.ts`:

- `PresetPortion` (исходная TOML-форма) — добавить `notes?: string`.
- `GemPortion` (скомпилированная форма, используется оверлеем) — добавить
  `notes?: string`, копируется без изменений из `PresetPortion`.

TOML-формат (`gems/<id>.toml`):

```toml
[[portion]]
quest = "a1q2"
take  = [ "Freezing Pulse" ]
buy   = [ "War Cry" ]
notes = "Свободный текст — что угодно."
```

## Парсинг/запись — `src/main/preset-store.ts`

- `parsePortion()` — читает `notes` из сырого TOML тем же способом, что и
  остальные строковые поля (`asString`).
- `writePreset()` — пишет `notes` в выходной TOML, только если поле непустое
  (по аналогии с `take`/`buy`, которые пишутся только когда `.length > 0`).

## Компиляция — `src/main/guide-loader.ts` (`parsePreset`)

- `notes` из исходной `PresetPortion` копируется в скомпилированный `GemPortion`.
- Текущая логика отбрасывает порцию целиком, если `portionSteps()` вернул пустой
  массив (`if (steps.length === 0) return`). Условие меняется: порция
  пропускается, только если у неё одновременно нет шагов (take/buy) **и** нет
  заметки. Порция с одними заметками (без take/buy) должна остаться видимой.

## Редактор — `src/renderer/src/settings/SettingsApp.tsx`

В блоке порции (там же, где сейчас `GemChips` "TAKE" и `BuyGemChips` "BUY"),
под ними — `<textarea>`, привязанная к `p.notes ?? ''`, при изменении обновляет
`d.portions[pi].notes = value || undefined` (пустая строка → поле убирается, как
и для остальных опциональных строковых полей в этом файле).

Новые строки i18n (`src/shared/i18n.ts`, ru + en): подпись поля (`portionNotesLabel`)
и placeholder (`portionNotesPlaceholder`).

## Оверлей — `src/renderer/src/App.tsx`

В блоке активной порции, сразу под `.portion-title`, до списка TAKE/BUY-шагов:

```tsx
{activePortion.notes && (
  <div className="notes portion-notes">
    <Markup text={activePortion.notes} />
  </div>
)}
```

Переиспользует существующий класс `.notes` (уже стилизован: `white-space: pre-wrap`,
`line-height`, приглушённый цвет) плюс модификатор `.portion-notes` под точечные
отступы, если понадобится. Инлайн-разметка (`{item|...}`, `{zone|...}` и т.д.)
работает как в любом другом тексте через `Markup`.

## Документация

Обновить таблицу `[[portion]]` в `docs/DATA-FORMATS.md` — добавить строку `notes`.

## Вне скоупа

- Zone-gems записи (`EntryRow`, тип "Награда"/"Покупка") — без изменений.
- Никаких новых kind-значений, никаких изменений `StepKind`/`GemEntry`.
