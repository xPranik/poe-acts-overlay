# Мини-таймер в шапке (цифры вместо иконки)

Дата: 2026-07-19

## Цель

Минималистичный режим таймера: пользователь запускает таймер, сворачивает панель
таймера, и вместо иконки `⏱` в шапке оверлея тикают цифры прошедшего времени.
Никаких новых окон и панелей — только замена содержимого существующей кнопки.

## Поведение

Кнопка таймера в `header-actions` (`src/renderer/src/App.tsx`):

| Состояние | Содержимое кнопки |
|---|---|
| Панель таймера открыта (`state.timer.visible === true`) | Иконка `⏱` с классом `active` — как сейчас |
| Панель свёрнута, таймер `idle` | Иконка `⏱` — как сейчас (показывать `0:00` нет смысла) |
| Панель свёрнута, таймер `running` | Цифры, тикают раз в секунду |
| Панель свёрнута, таймер `paused` | Цифры, статичные, приглушённые (CSS-класс) |
| Панель свёрнута, таймер `finished` | Цифры, статичное итоговое время |

- Формат времени: `mm:ss`, при часе и более `h:mm:ss`. Без десятых долей —
  в маленькой шапке они создают только визуальный шум.
- Клик по кнопке ведёт себя как раньше: `window.api.timerToggleVisible()`
  (открыть/закрыть панель таймера). Tooltip (`t.runTimerTitle`) не меняется.

## Реализация

- **Переиспользование логики.** `fmt()` и `liveElapsed()` сейчас локальные в
  `src/renderer/src/Timer.tsx` — экспортировать их оттуда (или вынести в мелкий
  хелпер `timer-format.ts`, если экспорт из файла компонента окажется грязным;
  решение по месту). Логика форматирования и подсчёта не дублируется.
- **Компонент.** Небольшой `MiniTimer` в `App.tsx` (или рядом): принимает
  `state.timer`, локальный тик через `setInterval` + `useState(now)` — тот же
  паттерн, что в `Timer.tsx`. Интервал активен только когда цифры видимы и
  статус `running`; в `paused`/`finished` — статичное значение без интервала.
- **CSS** (`src/renderer/src/styles.css`): для кнопки с цифрами —
  `font-variant-numeric: tabular-nums`, ширина по содержимому (чтобы цифры не
  «прыгали»), стиль шрифта согласован с временем в панели таймера; класс для
  приглушённого состояния `paused`.

## Ошибки и крайние случаи

- Новых IPC-каналов и состояния в main-процессе нет — всё берётся из уже
  приходящего `state.timer` (`accumulatedMs`, `runningSince`, `status`).
- Переход `running → paused` при свёрнутой панели просто останавливает тик.
- Сброс таймера (`idle`) при свёрнутой панели возвращает иконку.

## Проверка

1. `npm run typecheck`.
2. Ручная проверка через `scripts/fake-log.ts`: запустить таймер, свернуть
   панель — цифры тикают; пауза — цифры замирают и приглушаются; финиш —
   итоговое время; сброс — снова иконка; открытие панели — снова иконка.
